# 🤖 AI University Project: Reinforcement Learning

Проект обучения интеллектуальных агентов с подкреплением для двух сценариев:
- **CartPole**: Балансировка шеста на движущейся платформе
- **Maze**: Навигация в лабиринте к целевой точке

## ✨ Возможности

✅ Q-Learning алгоритм обучения с подкреплением  
✅ Сохранение и загрузка Q-таблиц  
✅ Возможность переучивания модели (сброс Q-таблицы)  
✅ MySQL база данных для отслеживания статистики  
✅ Web интерфейс с Server-Sent Events для потокового отображения обучения  
✅ Визуализация обучения с помощью Pygame  
✅ Логирование всех операций  
✅ Готовность к развертыванию на Railway  

## 📋 Требования

- Python 3.11+
- MySQL 5.7+ (или XAMPP)
- pip

## 🚀 Установка и локальный запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/Yange4e/AI_University_Project.git
cd AI_University_Project
```

### 2. Создание виртуального окружения
```bash
python -m venv .venv

# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Настройка базы данных

#### Локально (XAMPP):
1. Откройте XAMPP Control Panel
2. Запустите Apache и MySQL
3. Перейдите в phpMyAdmin: http://localhost/phpmyadmin
4. Создайте новую базу данных `ai_project`
5. Импортируйте `setup_db.sql`

Или выполните в MySQL:
```sql
SOURCE setup_db.sql;
```

### 5. Конфигурация окружения
```bash
cp .env.example .env
# Отредактируйте .env если нужно изменить параметры БД
```

### 6. Запуск приложения
```bash
python app.py
```

Приложение будет доступно по адресу: **http://localhost:5000**

## 🎓 Использование

### Web интерфейс

1. Откройте http://localhost:5000
2. Нажмите на кнопку обучения (CartPole или Maze)
3. Выберите количество эпизодов
4. Нажмите "Начать обучение"
5. Смотрите процесс в реальном времени

### Функции

- **Обучение**: Продолжает обучение с сохраненной Q-таблицей
- **Переучивание**: Сбрасывает Q-таблицу и начинает заново (параметр `reset=true`)
- **Статистика**: Отслеживает лучший результат и количество эпизодов

## 🚢 Развертывание на Railway

### Подготовка

1. Убедитесь, что все файлы залиты на GitHub:
```bash
git add .
git commit -m "Prepare for Railway deployment"
git push
```

2. Перейдите на https://railway.app
3. Нажмите **Start Project**
4. Выберите **Deploy from GitHub repo**
5. Авторизуйтесь и выберите `Yange4e/AI_University_Project`

### Добавление MySQL

1. В Railway нажмите **+ New**
2. Выберите **MySQL**
3. Railway создаст базу автоматически

### Переменные окружения

Установите в Railway:
```
FLASK_ENV=production
PORT=3000
```

MySQL переменные будут установлены автоматически:
- `MYSQLHOST`
- `MYSQLUSER`
- `MYSQLPASSWORD`
- `MYSQLDATABASE`
- `MYSQLPORT`

### Инициализация БД

После первого развертывания запустите в контейнере:
```sql
SOURCE setup_db.sql;
```

Или используйте Railway Plugin для выполнения SQL запросов.

## 📊 API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|---------|
| GET | `/` | Главная страница со статистикой |
| GET | `/train_view?type=cartpole&episodes=50&reset=false` | Обучение CartPole |
| GET | `/train_view?type=maze&episodes=50&reset=false` | Обучение Maze |
| GET | `/api/stats` | JSON со статистикой агентов |
| GET | `/health` | Health check для Railway |

### Параметры обучения

- `type`: `cartpole` или `maze` (обязательный)
- `episodes`: количество эпизодов (1-1000, по умолчанию 50)
- `reset`: `true` или `false` - сбросить ли Q-таблицу (по умолчанию false)

## 🔧 Параметры Q-Learning

### CartPole
- Learning Rate: 0.1
- Gamma (дисконтирование): 0.95
- Epsilon (исследование): 0.2 (уменьшается)
- Действия: [влево, вправо]

### Maze
- Learning Rate: 0.2
- Gamma (дисконтирование): 0.9
- Epsilon (исследование): 0.3 (уменьшается)
- Действия: [вверх, вниз, влево, вправо]

## 📊 Мониторинг и логирование

Все операции логируются с уровнями:
- 🔵 DEBUG: Детальная информация
- 🟢 INFO: Общая информация (✅ успех, 🎯 начало, 🏆 завершение)
- 🟡 WARNING: Предупреждения (⚠️)
- 🔴 ERROR: Ошибки (❌)

Логи выводятся в консоль и содержат временные метки.

## 🐛 Решение проблем

### Модель повторяет один и тот же маршрут

**Проблема**: После первого обучения агент повторяет только выученный маршрут.

**Решение**: Используйте параметр `reset=true` для переучивания:
- URL: `/train_view?type=maze&episodes=50&reset=true`

Или добавьте небольшой шум в Q-таблицу (это уже реализовано в коде).

### Ошибка подключения к MySQL

1. Убедитесь, что MySQL запущен
2. Проверьте параметры в `.env`
3. Убедитесь, что база данных `ai_project` создана
4. Запустите `setup_db.sql`

### Pygame не отображается на Railway

Pygame окна работают только локально. На Railway используется `render_mode="rgb_array"`.
